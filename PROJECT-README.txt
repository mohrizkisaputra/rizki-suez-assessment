## Objectives

Build a web-based IoT sensor monitoring application for command center using Angular with clean architecture.
The REST API just using static json files.
Include a README.md with clear plan about each features & setup instructions to get start and design decisions.

The focus of this test is on:

- Data Modeling
- Code Documentation & Readability
- Code Structure
- REST API Design & Consumption
- Testing Code

You can choose any domain such:

- Water monitoring
- Smart farming
- Energy monitoring
- etc.

What I would like to see one page dashboard that consist of

- map (5 sensors)
- pushpin and popup in the map
- several widgets component
- alert status notification in the map or on the widget

Each features, must be committed incrementally using Git, with meaningful message. Avoid a single large "final commit"

Deploy the application to a public accessible URL.

Upload a copy of the repository (including .git history) share it via Google Drive
